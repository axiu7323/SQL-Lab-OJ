# Codex 工作规则

## 1. 基本原则

Codex 每次只能执行一个任务文件。不得跳过文档直接改代码，不得大范围无边界重写项目。

## 2. 执行步骤

1. 阅读当前任务文件。
2. 阅读任务文件列出的规范文件。
3. 分析当前代码。
4. 输出修改计划。
5. 执行修改。
6. 自检是否符合验收标准。
7. 输出变更清单、风险点、未完成事项。

## 3. 禁止行为

- 修改任务范围之外的文件。
- 引入未经确认的新技术栈。
- 删除已有可运行代码。
- Controller 直接调用 Mapper。
- ApplicationService 直接调用 Mapper。
- Domain 依赖 Spring 或 MyBatis。
- 判题逻辑和 Controller 强耦合。
- 前端所有 API 混在一个文件。

## 4. 推荐提示词

```text
请严格按照 docs/09-tasks/xxx.md 执行任务。
执行前必须阅读任务文件中列出的所有规范文件。
本轮只允许执行当前任务，不允许修改任务范围之外的文件。
修改前先输出执行计划。
修改后输出变更文件清单、验收结果和未完成事项。
```
